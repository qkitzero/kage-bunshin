# Brainstorm スキル

Ideator と Analyst エージェントを協調させ、アイデアの発散と収束を一度に行うワークフローです。

## ワークフロー

### ステップ1: テーマの把握

ユーザーから提示されたテーマ・課題・問題を正確に理解する。曖昧な場合は、以下を確認してから進む：
- 対象領域（ビジネス、技術、クリエイティブなど）
- 制約条件（予算、期間、技術スタックなど）
- 求められる成果物の粒度

### ステップ1.5: 過去のNotebookコンテキスト取得

`NOTEBOOK_PATH` が設定されている場合、ユーザーのテーマに関連する過去のエントリを検索する：

1. `$NOTEBOOK_PATH/ideas/` 内をテーマのキーワードで Grep（タイトル、タグ、本文を対象）
2. マッチしたエントリを最大3件まで Read（フロントマター＋本文の先頭200文字）
3. 取得した内容を以下の形式でステップ2のエージェント指示に追加する：

```
## 過去の関連アイデア（Notebookより）
### [タイトル] (date: YYYY-MM-DD)
[本文の先頭200文字]
```

マッチが0件、または `NOTEBOOK_PATH` が未設定の場合はこのステップをスキップする。

### ステップ2: 発想フェーズ（Ideator）

`ideator` エージェントを呼び出し、以下の指示で5〜7個の多様なアイデアを生成させる：

**ideator への指示:**
```
テーマ: [ユーザーのテーマ]

以下の条件でアイデアを5〜7個生成してください：
- 各アイデアは異なるアプローチ・視点から発想すること
- 安全策だけでなく、大胆なアイデアも含めること
- 各アイデアに概要、背景・根拠、インパクト評価を付けること
- NOTEBOOK_PATH が設定されている場合、結果を $NOTEBOOK_PATH/ideas/ に保存すること
```

### ステップ3: 評価フェーズ（Analyst）

`analyst` エージェントを呼び出し、Ideator が生成した全アイデアを評価させる：

**analyst への指示:**
```
以下のアイデア群を評価してください：

[ideator の出力]

各アイデアについて以下の軸で1〜5のスコアを付け、総合評価してください：
- 実現可能性 (Feasibility)
- インパクト (Impact)
- 新規性 (Novelty)
- リスク (Risk: 低い方が高スコア)

さらに、各アイデアの強み・弱み・改善案を簡潔に述べてください。
NOTEBOOK_PATH が設定されている場合、評価結果を $NOTEBOOK_PATH/reviews/ に保存すること
```

### ステップ4: 統合・ランキング

Analyst の評価結果をもとに、以下の形式で最終出力を作成する：

```
## ブレインストーミング結果: [テーマ]

### 第1位: [アイデア名]
- **総合スコア**: X.X / 5.0
- **概要**: ...
- **評価**: 実現可能性 X | インパクト X | 新規性 X | リスク X
- **強み**: ...
- **改善案**: ...

### 第2位: [アイデア名]
...

### 第3位: [アイデア名]
...

### その他のアイデア
- [アイデア名]: 総合スコア X.X - [一行サマリ]

### 次のステップ
- 有望なアイデアに対して `/evaluate` で詳細評価を推奨
- プロジェクト化するなら `/plan-project` を推奨
```

## 出力形式

最終的にユーザーに提示するのは：
1. トップ3アイデアの詳細（スコア・評価付き）
2. その他のアイデアの一覧
3. 次のステップの提案
4. Notebook保存した場合はその旨を通知
