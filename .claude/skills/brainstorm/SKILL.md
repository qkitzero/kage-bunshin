# Brainstorm Skill

A workflow that coordinates the Ideator and Analyst agents to perform idea divergence and convergence in one session.

## Workflow

### Step 1: Understand the Theme

Accurately understand the theme, challenge, or problem presented by the user. If ambiguous, confirm the following before proceeding:
- Target domain (business, technology, creative, etc.)
- Constraints (budget, timeline, tech stack, etc.)
- Required granularity of deliverables

### Step 1.5: Retrieve Past Notebook Context

If `NOTEBOOK_PATH` is set, search for past entries related to the user's theme:

1. Grep `$NOTEBOOK_PATH/ideas/` for keywords from the theme (searching title, tags, and body)
2. Read up to 3 matching entries (frontmatter + first 200 characters of body)
3. Prepend the retrieved content to the agent instructions in Step 2 in this format:

```
## Related Past Ideas (from Notebook)
### [Title] (date: YYYY-MM-DD)
[First 200 characters of body]
```

Skip this step if zero matches or `NOTEBOOK_PATH` is not set.

### Step 2: Ideation Phase (Ideator)

Invoke the `ideator` agent with the following instructions:

**Instructions for ideator:**
```
Theme: [User's theme]

Generate 5-7 ideas under the following conditions:
- Each idea should come from a different approach/perspective
- Include bold ideas, not just safe ones
- Include summary, rationale, and impact assessment for each idea
- If NOTEBOOK_PATH is set, save results to $NOTEBOOK_PATH/ideas/
```

### Step 3: Evaluation Phase (Analyst)

Invoke the `analyst` agent to evaluate all ideas generated by the Ideator:

**Instructions for analyst:**
```
Evaluate the following ideas:

[ideator's output]

Score each idea on the following axes (1-5) and provide an overall assessment:
- Feasibility
- Impact
- Novelty
- Risk (lower risk = higher score)

Also briefly describe the strengths, weaknesses, and improvement suggestions for each idea.
If NOTEBOOK_PATH is set, save evaluation results to $NOTEBOOK_PATH/reviews/
```

### Step 4: Synthesis & Ranking

Based on the Analyst's evaluation, create the final output in this format:

```
## Brainstorming Results: [Theme]

### #1: [Idea Name]
- **Overall Score**: X.X / 5.0
- **Summary**: ...
- **Scores**: Feasibility X | Impact X | Novelty X | Risk X
- **Strengths**: ...
- **Improvement Suggestions**: ...

### #2: [Idea Name]
...

### #3: [Idea Name]
...

### Other Ideas
- [Idea Name]: Overall Score X.X - [One-line summary]

### Next Steps
- Recommend `/evaluate` for detailed assessment of promising ideas
- Recommend `/plan-project` for turning ideas into projects
```

## Output Format

Present the following to the user:
1. Top 3 ideas in detail (with scores and evaluation)
2. List of remaining ideas
3. Next step suggestions
4. Notebook save notification if applicable
