# Plan Project スキル

Researcher と Planner エージェントを協調させ、調査に裏付けられた実行可能なプロジェクト計画を作成するワークフローです。各エージェントが成果物を直接Notebookに保存します。

## ワークフロー

### ステップ1: 目標の把握

ユーザーから提示されたプロジェクトの目標・目的を正確に理解する。以下を明確にする：
- プロジェクトのゴール（何を達成したいか）
- スコープ（何を含み、何を含まないか）
- 既知の制約（期間、予算、人員、技術スタックなど）
- 成功の定義（どうなったら成功か）

曖昧な場合はユーザーに確認してから進む。

### ステップ1.5: 過去のNotebookコンテキスト取得

`NOTEBOOK_PATH` が設定されている場合、プロジェクト目標に関連する過去のエントリを検索する：

1. `$NOTEBOOK_PATH/plans/` 内を目標のキーワードで Grep（タイトル、タグ、本文を対象）
2. さらに `project:` フロントマターが一致するエントリも検索する
3. マッチしたエントリを最大3件まで Read（フロントマター＋本文の先頭200文字）
4. 取得した内容を以下の形式でステップ2・3のエージェント指示に追加する：

```
## 過去の関連計画（Notebookより）
### [タイトル] (date: YYYY-MM-DD)
[本文の先頭200文字]
```

マッチが0件、または `NOTEBOOK_PATH` が未設定の場合はこのステップをスキップする。

### ステップ2: リサーチフェーズ（Researcher）

`researcher` エージェントを呼び出し、計画に必要な背景知識を収集させる：

**researcher への指示:**
```
以下のプロジェクトを計画するための事前調査を行ってください：

【プロジェクト目標】
[ユーザーのプロジェクト目標]

調査項目：
1. 類似プロジェクトの事例とそこから得られる教訓
2. ベストプラクティスと推奨アプローチ
3. よくある落とし穴・失敗パターン
4. 必要な技術・ツール・リソースの概要
5. 想定すべきリスクと対策

簡潔なリサーチサマリーとして出力してください。
NOTEBOOK_PATH が設定されている場合、リサーチ結果を $NOTEBOOK_PATH/research/ に保存すること
```

### ステップ3: 計画フェーズ（Planner）

`planner` エージェントを呼び出し、リサーチ結果を踏まえた詳細計画を作成させる：

**planner への指示:**
```
以下のプロジェクトの詳細計画を作成してください：

【プロジェクト目標】
[ユーザーのプロジェクト目標]

【制約条件】
[ユーザーが提示した制約]

【リサーチ結果】
[researcher の出力]

以下の要素を含む計画を作成してください：

1. **フェーズ分割**: プロジェクトを3〜5のフェーズに分割
2. **マイルストーン**: 各フェーズの完了条件
3. **タスク分解**: 各フェーズ内の具体的タスク（実行可能な粒度）
4. **依存関係**: タスク間の前後関係
5. **リスクと対策**: 主要リスク3〜5個とその緩和策
6. **成功基準**: 測定可能な成功指標
```

### ステップ4: 計画書の整形

planner の出力をもとに、以下の形式で最終出力を作成する：

```
## プロジェクト計画: [プロジェクト名]

### 概要
- **目標**: ...
- **スコープ**: ...
- **想定期間**: ...
- **成功基準**: ...

---

### フェーズ 1: [フェーズ名] (期間目安)
**マイルストーン**: [完了条件]

| # | タスク | 依存 | 見積 | 備考 |
|---|--------|------|------|------|
| 1.1 | ... | - | ... | ... |
| 1.2 | ... | 1.1 | ... | ... |
| ... | ... | ... | ... | ... |

### フェーズ 2: [フェーズ名] (期間目安)
...

---

### リスク管理

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| ... | 高/中/低 | 高/中/低 | ... |

### リサーチからの教訓
- [類似プロジェクトからの学び]
- [避けるべき落とし穴]

### 次のステップ
- フェーズ1の最初のタスクから着手
- 定期的に `/retrospect` で振り返りを推奨
```

ステップ3のplanner指示に、Notebook保存の指示を含める：
```
NOTEBOOK_PATH が設定されている場合、計画書を $NOTEBOOK_PATH/plans/ に保存すること
```

`NOTEBOOK_PATH` が未設定の場合は、保存をスキップし結果を会話内に出力するのみとする。

## 出力形式

最終的にユーザーに提示するのは：
1. プロジェクト概要
2. フェーズ別の詳細計画（タスク・依存関係・見積付き）
3. リスク管理表
4. リサーチからの教訓
5. 次のステップ
6. Notebook保存した場合はその旨を通知
